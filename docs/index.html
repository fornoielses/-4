Навигация: →/← или клик

Цель и измеримые результаты

  • Функция: встроенный SUS‑опрос после завершения курса.
  • Где: веб‑компонент «Личный кабинет → Мои курсы → Оценить».
  • Артефакты: Issue, ветка, Pull Request, тесты, отчёт, презентация.

Критерии приёмки:

  • UI доступен только после прохождения ≥ 80% курса.
  • Ответы сохраняются и агрегируются (среднее SUS, распределение).
  • Экспорт в CSV/JSON и просмотр отчёта наставником.
  • Покрытие unit‑тестами ≥ 70% для нового кода.

Стратегия ветвления: GitHub Flow (ветка на фичу → PR → ревью → main)

# 1) Создаём Issue с шаблоном
Title: feat(sus): модуль SUS‑опроса
Labels: feature, frontend, backend, analytics, priority:high
Acceptance: см. критерии на слайде «Цель»

# 2) Ветка под фичу
$ git checkout -b feat/sus-survey
# 3) Коммиты по Conventional Commits
$ git commit -m "feat(sus): UI формы и валидация"
$ git commit -m "feat(sus): REST эндпоинт /api/sus {post,get}"
$ git commit -m "test(sus): unit + e2e"
# 4) Push и Pull Request с шаблоном
$ git push -u origin feat/sus-survey
  • Правила: только fast‑forward merge через PR, защита main.
  • Обязательные проверки: lint, unit, build, e2e, Lighthouse ≥ 90.
  • Code owners: frontend и platform — 2 апрува минимум.
  • Preview: Deploy Preview (Vercel/Pages) на каждый commit.

Архитектура решения (упрощённо)

Фронтенд: компонент SusSurvey.tsx, контекст курса, форма 10 вопросов, подсчёт метрики (0–100), доступность (A11y).

Бэкенд API: POST /api/sus (сохранить ответы), GET /api/sus/summary?courseId=….

Хранилище: таблицы sus_responses, sus_summary, шардирование по courseId.

// Псевдокод расчёта SUS
function calcSUS(answers /* 1..5 */){
  let pos = [1,3,5,7,9]; let neg=[2,4,6,8,10]
  let s = pos.reduce((t,i)=>t+(answers[i]-1),0)
        + neg.reduce((t,i)=>t+(5-answers[i]),0)
  return s * 2.5 // шкала 0..100
}

Структура отчёта

  1. Введение: цель, контекст, ограничения.
  2. Выбор стратегии ветвления и правила репозитория.
  3. Проектирование фичи: API, схема данных, UX‑макет.
  4. Реализация: коммиты, Pull Request, CI‑пайплайн.
  5. Тестирование и результаты метрик (SUS, Perf, A11y).
  6. Риски, ретроспектива, рекомендации.
  7. Приложения: ссылки, логи CI, скриншоты, экспорт CSV.

Совет: распечатайте в PDF (Ctrl/Cmd+P) — слайды автоматически раскрываются списком.
